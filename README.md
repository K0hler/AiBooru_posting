# AIBooru Auto-Poster

Автоматическая загрузка AI-сгенерированных изображений на [aibooru.online](https://aibooru.online).

## Возможности

- Сканирование локальной папки на наличие новых изображений (PNG, JPG, WEBP)
- Автоматическое тегирование через WD Tagger v3 (локально, без API)
- Извлечение метаданных Automatic1111 из PNG и добавление в AI_metadata
- Двухшаговая загрузка через AIBooru API
- Дедупликация по SHA256 — повторно не загружает
- Автоопределение рейтинга (general/sensitive/questionable/explicit)

## Установка

```bash
git clone <repo-url>
cd AiBooru_posting
pip install -r requirements.txt
```

## Настройка

Создайте файл `.env` в корне проекта:

```
AIBOORU_LOGIN=ваш_логин
AIBOORU_API_KEY=ваш_api_ключ
IMAGES_DIR=путь/к/папке/с/изображениями
```

API-ключ генерируется в настройках профиля на aibooru.online.

## Использование

```bash
python main.py
```

Пример вывода:

```

Загрузка WD Tagger... ОК
Найдено 12 изображений, 3 новых
[1/3] image_001.png -- 24 тегов, rating: s -- загружено OK
[2/3] image_002.png -- 2 тегов -- пропущено (недостаточно тегов)
```

При первом запуске скачивается модель WD Tagger (~467MB).

## Как это работает

1. Скрипт проверяет подключение к AIBooru и валидность учётных данных
2. Сканирует папку `IMAGES_DIR` и находит новые файлы (сравнивает хеши с `posted.json`)
3. Спрашивает сколько изображений загружать (Если вы не топ юзер, то у вас есть лимиты на количество постов, смотрите в профиле)
4. Каждое изображение прогоняется через WD Tagger — получаем теги и рейтинг
5. Валидация: минимум 5 тегов, рейтинг определён
6. Извлекаются метаданные A1111 из PNG (prompt, model, seed и т.д.)
7. Загрузка на AIBooru: файл → ожидание обработки → создание поста
8. Успешно загруженные файлы помечаются в `posted.json`
